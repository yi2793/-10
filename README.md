## 1 Markdown 高级使用概述
Markdown[^1] 是轻量级标记语言，基础语法可满足简单排版需求，但在技术文档编写、团队协作、理工科内容创作中，需掌握高级功能突破使用边界。本手册涵盖页内跳转、脚注、数学公式等十大核心技能，遵循**语义化排版规范**，让文档从「能用」升级为「好用、好看、专业」。

遵循排版原则：中文与英文/数字之间加一个空格，如编写 Markdown 文档需搭配 VS Code 编辑器；标题层级严格遵循 `#-##-###` 规则，不跳级；所有标题前后留空行，段落间适当留白，提升阅读体验。

## 2 核心功能实操指南
### 2.1 页内跳转：长文档的「传送门」
长文档（超1000字）需设置页内跳转，避免读者滚动鼠标查找信息，核心语法为**「设置终点（标题）+ 建立起点（链接）」**。
- 终点：确定目标标题，如 `### 2.1 页内跳转`
- 起点：`[跳转文字](#标题名)`，如 `[查看页内跳转技巧](#21-页内跳转)`
> 注意：英文标题有空格时，跳转链接需将空格替换为 `-`，如 `## My Skills` 对应 `(#my-skills)`。

### 2.2 脚注与注释：让文档更严谨
正文避免冗余解释，补充说明放在页脚，核心语法为**「正文标记 + 文末解释」**。
- 正文标记：`需要解释的内容[^编号]`，如 `开源协议遵循 MIT[^2]`
- 文末解释：`[^编号]: 具体说明`，所有脚注统一放在手册末尾附录。

### 2.3 数学公式：理工科内容刚需
无需插入图片公式，直接用代码编写，支持行内公式和独立块公式，核心符号：`^` 代表次方，`_` 代表下标。
- 行内公式：一对 `$` 包裹，如 $S=πr^2$（圆的面积公式）
- 独立块公式：两对 `$$` 包裹，如常用的勾股定理：
$$a^2 + b^2 = c^2$$

### 2.4 HTML 增强：突破 Markdown 极限
Markdown 原生语法无法实现的样式，可借用 HTML 标签，**不滥用**，仅在必要时使用。
- 文字改色：`<span style="color:red">极其重要</span>` → <span style="color:red">极其重要</span>
- 文字居中：`<center>Markdown 高级技术手册 核心要点</center>`

### 2.5 Emoji 魔力：让文档有温度
用 Emoji 标注重点、区分状态，缓解纯文字枯燥，输入语法为**两个冒号包裹英文单词**，可在 `gitmoji.dev` 查找对应图标。
- 常用场景：✅ 完成、⚠️ 注意、📌 重点、🔧 实操、📚 知识点

### 2.6 特殊符号转义：避免符号「撞车」
当需要在文档中显示 Markdown 排版符号（如 `#`、`**`、`[]`），且不触发其功能时，在符号前加**反斜杠 \** 转义。
示例：\*\*这段文字两边有星号，但它不加粗\*\* → **这段文字两边有星号，但它不加粗**
其他示例：\# 这是普通井号，不是标题；\_ 这是普通下划线，不是斜体。

### 2.7 相对路径与图床：让图片永不丢失
团队协作（如 GitHub）中，图片需放在统一文件夹，用**相对路径**引用，确保不同环境正常显示。
核心语法：`![图片描述](./images/图片文件名.格式)`
jpeg 格式示例：`![Markdown 学习路径](./images/markdown_learn.jpeg)`
> 注意：`./` 代表当前文档所在文件夹，需先建立 `images` 文件夹存放所有图片。

### 2.8 任务列表：高效管理协作进度
自带复选框的列表，适合个人学习规划、团队任务管理，核心语法需**保留中括号两边的空格**。
- 未完成：`- [ ] 任务内容`
- 已完成：`- [x] 任务内容`
具体示例见「3 学习与实践规划」章节。

## 4 常用公式与符号速查
### 4.1 理工科常用行内公式
- 圆的周长：$C=2πr$
- 面积公式（正方形）：$S=a^2$
- 一次函数：$y=kx+b$

### 4.2 常用转义符号示例
| 需显示的符号 | 转义语法 | 渲染结果 |
|--------------|----------|----------|
| **           | \*\*     | **       |
| #            | \#       | #        |
| _            | \_       | _        |
| [ ]          | \[ \]    | [ ]      |
